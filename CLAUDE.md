## Project Overview

AI List is a curated directory of AI products, tools, platforms, models, and infrastructure. It's a static site built with **Hugo** using Content Adapters to generate pages from YAML data files. Product data lives as individual YAML files in `data/products/`, and Hugo generates individual product profile pages, taxonomy pages, and a filterable list page.

Site: https://spacesitenet.github.io/ailist/

## Commands

```bash
# Development
bun run dev              # Start Hugo dev server
bun run build            # Build static site (hugo --minify)
bun run preview          # Preview built site

# Data tools
bun run check            # Validate all YAML files against Zod schema
bun run add              # Interactive CLI to create a new product YAML

# Linting
bun run lint             # Check TypeScript with Biome
bun run lint:fix         # Auto-fix lint issues
```

## Architecture

### Data Flow

Individual product YAML files (`data/products/*.yaml`) → Hugo Content Adapter (`content/products/_content.gotmpl`) generates pages from data → Hugo templates render product profiles, list pages, taxonomy pages → TypeScript handles client-side filtering and sorting.

### No Build Pipeline

Unlike the previous Astro+React setup, there is no preprocessing step. `hugo --minify` does everything:

- Reads YAML data files directly
- Generates all HTML pages via Content Adapters
- Compiles TypeScript via `js.Build` (esbuild)
- Processes Tailwind CSS via `css.TailwindCSS`
- Generates sitemap, robots.txt, llms.txt

### Frontend Architecture

- **Hugo** generates all HTML server-side (SSG) with Go templates
- **Content Adapters** (`content/products/_content.gotmpl`) generate pages from YAML data — no markdown content files
- **Tailwind v4** installed via Bun, processed by Hugo's `css.TailwindCSS` pipe
- **TypeScript** compiled by Hugo's `js.Build` (esbuild) — handles client-side filtering, sorting, URL sync
- **Zod** (`scripts/lib/schema.ts`) validates product data at development time

### Key Template Flow

`layouts/index.html` (homepage = product list) → `layouts/partials/filters.html` (filter controls) → `layouts/partials/product-row.html` (table rows with data-* attributes) → `assets/ts/filter.ts` (FilterEngine toggles row visibility) → `assets/ts/url-sync.ts` (syncs state to URL params)

### Product Pages

- `/products/` — Filterable product table (`layouts/products/list.html`)
- `/products/{slug}/` — G2-style product profile (`layouts/products/single.html`)
- `/categories/{name}/` — Products by category (`layouts/categories/term.html`)
- `/platforms/{name}/` — Products by platform (`layouts/platforms/term.html`)

### Product Data Schema

Each product YAML in `data/products/` has normalized fields: `name`, `slug`, `description`, `website`, `year_founded`, `headquarters` (always `string[]`), `open_source`, `self_signup`, `discontinued`, `has_logo`, `categories`, `platforms` (top-level, always `string[]`), `models` (pricing/delivery), `stats`, `notes`, and future fields (`features`, `integrations`, `target_audience`, etc.). See `scripts/lib/schema.ts` for the full Zod schema.

### SEO

- **Schema.org**: WebSite+SearchAction, BreadcrumbList, SoftwareApplication (products), ItemList (list pages)
- **OG/Twitter Cards**: Product logos used as OG images for product pages
- **Sitemap**: Auto-generated by Hugo
- **LLM-friendly**: `/llms.txt` and `/llms-full.txt` following llmstxt.org spec
- **UTM tracking**: All outbound links include `utm_source=ailist&utm_medium=referral&utm_campaign=...`

## Adding a Product

1. Create `data/products/{slug}.yaml` (or use `bun run add`)
2. Optionally add logo to `logos/originals/dark/` or `logos/originals/light/`, then process to `static/assets/logos/{slug}.webp`
3. Run `bun run check` to validate
4. Commit the YAML file (and logo if added)

## CI/CD

GitHub Actions (`.github/workflows/deploy.yml`) on push to `main`:

1. Installs Bun + Hugo binary
2. `bun install` + `hugo --minify`
3. Deploys to GitHub Pages

No auto-commit of generated files. No preprocessing steps.

## Key Files

| File | Purpose |
|------|---------|
| `hugo.toml` | Site config, taxonomies, output formats |
| `data/products/*.yaml` | Source of truth — one YAML per product |
| `content/products/_content.gotmpl` | Content adapter generating pages from data |
| `layouts/products/single.html` | Product profile page template |
| `layouts/products/list.html` | Filterable product table template |
| `layouts/index.html` | Homepage (renders product list) |
| `assets/ts/filter.ts` | FilterEngine — client-side filtering |
| `assets/ts/main.ts` | Entry point — wires up all interactivity |
| `assets/css/main.css` | Tailwind v4 CSS with theme tokens |
| `scripts/lib/schema.ts` | Zod schema for normalized YAML |
| `scripts/check.ts` | Validates all YAML against schema |
| `biome.json` | Biome linter configuration |
